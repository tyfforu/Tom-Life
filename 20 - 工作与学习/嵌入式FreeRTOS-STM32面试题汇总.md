# 嵌入式 FreeRTOS STM32 面试题汇总\n\n> 来源：[知乎 - 嵌入式Free RTOS单片机STM32面试题汇总](https://zhuanlan.zhihu.com/p/1895104636026877368)\n> DeepMeet一对一 · 94 赞\n\n---\n\n## 一、FreeRTOS 主要知识点\n\n### 1. 核心概念\n\n- **任务（Task）**：FreeRTOS 的基本执行单元，具有运行态、就绪态、阻塞态、挂起态、删除态\n- **任务调度**：支持抢占式、时间片轮转、合作式调度\n- **内核对象**：队列、信号量、互斥量、事件组、任务通知，用于任务间通信和同步\n\n### 2. 任务管理\n\n- 创建任务：`xTaskCreate` 或静态创建\n- 删除任务：`vTaskDelete`\n- 任务优先级：优先级越高，越先获得 CPU\n\n### 3. 内存管理\n\n- 静态和动态内存分配\n- 动态内存：`pvPortMalloc` / `vPortFree`\n\n### 4. 同步与通信\n\n- **队列**：任务间 / 中断与任务间的消息传递\n- **信号量**：二值信号量 + 计数型信号量，用于同步和互斥\n- **互斥量**：保护共享资源，可避免优先级反转（优先级继承）\n- **事件组**：任务间同步\n- **任务通知**：轻量级任务间通信\n\n### 5. 低功耗支持\n\n- **Tickless 模式**：空闲时关闭滴答定时器，进入低功耗，事件触发时唤醒\n- **空闲任务回调**：`vApplicationIdleHook` 中调用 `enterLowPowerMode()`\n\n### 6. 移植与配置\n\n- 通过 `FreeRTOSConfig.h` 配置系统功能（任务优先级、调度策略、滴答频率）\n- 根据硬件平台选择内存管理方案\n\n---\n\n## 二、常见面试题\n\n### 基础概念\n\n**FreeRTOS 任务状态有哪些？**\n- Ready（就绪态）、Running（运行态）、Suspended（挂起态）、Blocked（阻塞态）\n\n**调度方式有哪些？**\n- 时间片轮转：同优先级任务轮流执行（默认 1ms）\n- 抢占式：高优先级任务强制打断低优先级任务\n- 协作式：`vTaskDelay`、信号量、互斥量\n\n**内核对象有哪些？**\n`croutine.c`（协程）、`event_groups.c`（事件组）、`list.c`（链表）、`queue.c`（队列）、`stream_buffer`（流缓冲）、`tasks.c`（任务）、`timers.c`（软件定时器）、`heap`（内存管理）\n\n### 任务管理\n\n**如何创建和删除任务？**\n```c\n// 动态创建\nxTaskCreate(taskFunc, "TaskName", stackSize, NULL, priority, &taskHandle);\n// 删除\nvTaskDelete(taskHandle);\n```\n\n**如何避免优先级反转？**\n使用互斥量 + 优先级继承机制\n\n### 同步与通信\n\n**信号量 vs 互斥量**\n- 信号量：同步 + 互斥，可能发生优先级反转，可计数\n- 互斥量：专用于保护共享资源，可避免优先级反转（优先级继承）\n\n**队列使用**\n```c\n// 创建\nQueueHandle_t xQueue = xQueueCreate(10, sizeof(int));\n// 发送\nxQueueSend(xQueue, &value, portMAX_DELAY);\n// 接收\nxQueueReceive(xQueue, &value, portMAX_DELAY);\n```\n\n### 内存管理\n\n**5 种 Heap 方案**\n- `heap_1`：只分配，不释放（简单，嵌入式常用）\n- `heap_2`：可释放，但不对齐\n- `heap_3`：封装 C 库 malloc/free\n- `heap_4`：可释放 + 自动整理碎片（推荐）\n- `heap_5`：支持跨内存区，heap_4 功能 + 多堆\n\n**避免内存泄漏**\n1. 选 heap_4 或 heap_5，避开 heap_1\n2. `vTaskDelete` 后手动释放 `pvPortMalloc` 分配的内存\n3. `vQueueDelete()` / `vSemaphoreDelete()` 释放内核对象\n4. 避免碎片化：预分配资源，减少频繁动态分配\n5. 用 `pvPortMalloc` / `vPortFree` 替代标准 malloc/free\n6. 启用 `configCHECK_FOR_STACK_OVERFLOW` 检测栈溢出\n7. `uxTaskGetStackHighWaterMark()` 监测栈使用\n\n### 低功耗\n\n**Tickless 工作流程**\n1. 空闲时计算下一次调度所需时间\n2. 关闭滴答定时器，进入低功耗模式\n3. 定时器中断唤醒，继续调度\n\n### 移植\n\n**移植步骤**\n1. 复制 FreeRTOS 源码（.c / .h）\n2. 移植 `FreeRTOSConfig.h`\n3. 编译解决报错（头文件路径等）\n\n### 性能优化（10 条）\n\n1. 减少中断延迟：ISR 保持简短，复杂操作推迟到任务\n2. 任务优先级设计：避免任务饥饿，合理分配\n3. 简化任务栈：减小栈大小，`configSTACK_DEPTH_TYPE` 调整\n4. 选对 heap：heap_4 / heap_5\n5. 减少动态分配：实时性要求高的任务用静态分配\n6. 调节 Tick 频率：`configTICK_RATE_HZ` 按需设置\n7. 定时器合并：减少中断次数\n8. 减少上下文切换：合理设计 CPU 占用时间\n9. 协作调度替代抢占：非严格实时场景\n10. 轻量级同步：优先用任务通知而非信号量\n\n---\n\n## 三、FreeRTOS 启动流程\n\n1. 初始化调度器\n2. 启动调度器\n3. 创建空闲任务\n4. 开启任务调度\n\n---\n\n## 四、任务状态监控\n\n1. `eTaskGetState(TaskHandle_t)` — 返回 eRunning/eReady/eBlocked/eSuspended/eDeleted\n2. 任务通知：`xTaskNotifyGive` + `ulTaskNotifyTake`\n3. 栈监测：`uxTaskGetStackHighWaterMark()`（接近 0 = 栈溢出风险）\n4. `vTaskGetRunTimeStats()` / `vTaskList()` — 任务资源占用\n5. 外部工具：**FreeRTOS+Trace**（可视化时间轴）、**SEGGER SystemView**（实时监控）\n6. 日志：`vApplicationIdleHook` 中定期输出系统状态\n7. 软/硬件定时器定期轮询任务状态\n\n---\n\n## 五、单片机经典面试题\n\n### GPIO（STM32 8 种模式）\n\n- 浮空输入 / 上拉输入 / 下拉输入 / 模拟输入\n- 开漏输出 / 开漏复用输出 / 推挽输出\n\n**推挽 vs 开漏**\n- 推挽：输出高低电平，驱动能力强（LED、继电器）\n- 开漏：只拉低，需上拉电阻，可实现线与逻辑（I2C 总线）\n\n### 通信协议对比\n\n| 协议 | 类型 | 线数 | 特点 |\n|---|---|---|---|\n| SPI | 全双工 | 4 | 快，无流控无应答 |\n| I2C | 半双工 | 2 | 多主控，支持总线共享 |\n| UART | 全双工 | 2 | 无时钟，距离远，有检错 |\n| CAN | 半双工 | 2（H/L差分） | 高可靠，汽车/工控 |\n\n### 无线通信\n\n- **BLE（蓝牙）**：2.4GHz，低功耗，近距离\n- **WiFi**：802.11b，2.4GHz，100m 覆盖\n- **NFC**：13.56MHz，双向识别\n- **Zigbee**：2.4GHz，低速低功耗，复杂拓扑\n\n### IoT 协议对比\n\n| 协议 | 传输层 | 适用场景 |\n|---|---|---|\n| MQTT | TCP | 物联网消息发布/订阅 |\n| CoAP | UDP | 低资源设备、低功耗网络 |\n| HTTP | TCP | Web 通信，开销大 |\n\n### 其他\n\n- **中断过程**：保护现场 → 执行 ISR → 恢复现场 → 继续执行\n- **TCP 三次握手**：SYN → SYN+ACK → ACK\n- **TCP 四次挥手**：FIN → ACK → FIN → ACK\n- **定时器作用**：计数、延时、输入捕获、输出比较（PWM）、中断生成\n\n---\n\n## 六、大数据包帧解析流程\n\n1. 初始化环形缓冲区\n2. 串口 ISR 分段接收，检测帧头和帧长\n3. 收满后用 CRC/校验和验证完整性\n4. 解析数据内容\n5. 校验失败则丢弃，重置缓冲区\n\n---\n\n## 七、FreeRTOS vs RT-Thread\n\n| | FreeRTOS | RT-Thread |\n|---|---|---|\n| 定位 | 轻量级内核，超小体积 | 功能丰富，含软件包管理器 |\n| 内存占用 | 极低 | 较多 |\n| 内存管理 | 简单静态/小块动态 | 完整堆和栈管理 |\n| 适用场景 | 资源极度受限的小系统 | 需要 GUI、网络等丰富功能的系统 |\n\n---\n\n## 八、裸机软件定时器实现\n\n利用硬件定时器中断触发全局计数器，每个软定时器维护倒计时。中断中轮询所有定时器，减计数，到零则触发任务。\n\n管理 100 个任务：链表/数组存储 + 定时器轮询。可用**时间轮（Timing Wheel）**算法优化查找效率。\n
