## **一、vi/vim 三种核心模式（必考基础）**
| 模式 | 进入方式 | 作用 | 退出方式 |
|------|----------|------|----------|
| **普通模式** | 启动 vim 或按 `Esc` | 执行命令、移动光标 | 按 `i`/`a`/`o` 进入插入模式 |
| **插入模式** | 普通模式下按：<br>• `i`（当前光标前插入）<br>• `a`（当前光标后插入）<br>• `o`（下一行插入） | 编辑文本内容 | 按 `Esc` 返回普通模式 |
| **命令模式** | 普通模式下按 `:` | 执行保存、退出、查找等命令 | 执行命令后自动返回普通模式 |

> **考试重点**：  
> - 启动 vim 后默认是**普通模式**，不能直接输入文字  
> - 按 `Esc` 是**万能返回键**，可从任何模式回到普通模式

---

## **二、光标移动命令（普通模式下）**
### **基础移动**
| 命令 | 作用 | 记忆技巧 |
|------|------|----------|
| `h` `j` `k` `l` | 左、下、上、右移动 | `j` 向下（像箭头↓），`k` 向上（像箭头↑） |
| `w` | 跳到下一个单词开头 | word 的首字母 |
| `b` | 跳到上一个单词开头 | backward |
| `0`（数字零） | 跳到行首 | zero = 开头 |
| `$` | 跳到行尾 | 美元符号在行尾 |

### **快速跳转**
| 命令 | 作用 | 考试场景 |
|------|------|----------|
| `gg` | 跳到文件第一行 | 快速定位文件开头 |
| `G` | 跳到文件最后一行 | 查看文件结尾 |
| `5G` | 跳到第 5 行 | 精确跳转到指定行 |
| `Ctrl + f` | 向下翻页 | 查看大文件 |
| `Ctrl + b` | 向上翻页 | 查看大文件 |

## **三、文本编辑操作（高频考点）**
### **插入与删除**
| 命令 | 作用 | 示例 |
|------|------|------|
| `i` | 在当前光标前插入 | 编辑配置文件时追加内容 |
| `a` | 在当前光标后插入 | 在行尾添加配置项 |
| `o` | 在下一行插入新行 | 添加新配置段落 |
| `x` | 删除当前字符 | 删除单个字符 |
| `dw` | 删除当前单词 | 删除配置项 |
| `dd` | 删除当前行 | 删除整行配置 |
| `d$` | 删除从光标到行尾 | 删除行尾多余内容 |

### **复制与粘贴**
| 命令 | 作用 | 组合技巧 |
|------|------|----------|
| `yy` | 复制当前行 | `5yy` 复制 5 行 |
| `p` | 粘贴到光标后 | `P` 粘贴到光标前 |
| `yw` | 复制当前单词 | 配合 `p` 快速重复单词 |

> **考试避坑**：  
> - 复制/粘贴必须在**普通模式**下操作，插入模式下按 `p` 会直接输入字母 'p'  
> - `dd` + `p` = 剪切粘贴

## **四、查找与替换（必考操作）**
### **查找文本**
| 命令 | 作用 | 考试示例 |
|------|------|----------|
| `/error` | 向下查找 "error" | 在日志文件中查找错误 |
| `?warning` | 向上查找 "warning" | 反向查找警告信息 |
| `n` | 跳到下一个匹配项 | 连续查找相同内容 |
| `N` | 跳到上一个匹配项 | 反向查找 |

### **替换文本**
| 命令 | 作用 | 考试场景 |
|------|------|----------|
| `:s/old/new` | 替换当前行第一个 "old" | 修正拼写错误 |
| `:s/old/new/g` | 替换当前行所有 "old" | 批量修改配置项 |
| `:%s/old/new/g` | 替换整个文件所有 "old" | 全局配置更新 |
| `:%s/old/new/gc` | 替换前确认（y/n） | 安全替换（考试高频） |

> **考试重点**：  
> - 末尾加 `g` = global（全局替换）  
> - 末尾加 `c` = confirm（确认替换），考试中要求安全操作时必须使用

---

## **五、保存与退出（关键操作）**
| 命令 | 作用 | 使用场景 |
|------|------|----------|
| `:w` | 保存文件 | 修改配置后保存 |
| `:q` | 退出（无修改时） | 查看文件后退出 |
| `:wq` 或 `:x` | 保存并退出 | 正常编辑完成 |
| `:q!` | 强制退出不保存 | 误操作后放弃修改 |
| `:w filename` | 另存为新文件 | 保存副本 |

> **考试避坑**：  
> - **`:wq!`**（带感叹号）用于强制保存只读文件（如 `/etc/passwd` 需要权限）  
> - 未保存时直接 `:q` 会提示 "No write since last change"，必须用 `:q!` 强制退出

## **六、省初赛高频考题类型**
### **1. 场景题：修改配置文件**
> **题目**：使用 vim 编辑 `/etc/nginx/nginx.conf`，在 server 块中添加 `listen 8080;` 配置。  
> **操作步骤**：  
 ```bash
 vim /etc/nginx/nginx.conf
 # 普通模式下：/server  # 查找 server 块
 # 按 n 跳到目标位置
 # 按 o 在下一行插入
 listen 8080;
 # 按 Esc，输入 :wq 保存退出
 ```

### **2. 命令填空题**
> **题目**：在 vim 中，从普通模式进入插入模式的命令是 ___（填一个字母）。  
> **答案**：`i`（或 `a`、`o`，但 `i` 最常用）

### **3. 判断题**
> **题目**：在 vim 的插入模式下，按 `dd` 会删除当前行。  
> **答案**：❌（`dd` 必须在普通模式下执行）

## **七、备考建议（针对考试）**
1. **必练操作**：  
   - 模式切换（普通 ↔ 插入）  
   - 保存退出（`:wq` vs `:q!`）  
   - 行首行尾跳转（`0` 和 `$`）  
   - 全局替换（`:%s/old/new/g`）

2. **考试技巧**：  
   - 遇到不会的操作，先按 `Esc` 回到普通模式  
   - 修改重要文件前，先用 `:w backup.conf` 备份  
   - 不确定命令时，用 `:help 命令` 查看帮助（考试环境通常允许）

3. **易错点总结**：  
   | 错误操作 | 正确做法 | 后果 |
   |----------|----------|------|
   | 插入模式下按 `:wq` | 先按 `Esc` 回普通模式 | 输入 ":wq" 作为文本 |
   | 用 `Ctrl + C` 代替 `Esc` | 严格按 `Esc` | 可能中断操作 |
   | 忘记保存直接 `:q` | 先 `:w` 再 `:q` | 丢失所有修改 |
