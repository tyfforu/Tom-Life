# STM32 "邪修"学习法：GPIO与LED闪烁

> 来源：B站 @A8（基于江协STM32课程）
> 
> 核心理念：以输出形式学习，用"工地比喻"理解寄存器操作

---

## 一、学习理念

### 为什么做这套视频？

- 以**新手视角**学习STM32
- 很多人学完后**无法默写程序**
- **输出式学习**：把知识讲出来才能记得牢
- **邪修方式**：用通俗比喻理解底层逻辑

### 学习建议

> 先学会**如何使用**，再深入了解底层逻辑。

## 二、核心比喻：工地模型

| 概念 | 比喻 | 说明 |
|------|------|------|
| **CPU** | 甲方 | 发起需求的人 |
| **APB2** | 工头老王 | 所有GPIO都挂载在APB2下 |
| **寄存器** | 工单 | 记载：找谁干活、怎么干、速度 |
| **驱动器** | 大型机械 | 需要大电压时的辅助设备 |
| **GPIO引脚** | 工人/班组 | GPIOA-GPIOE，每组16个引脚 |

**工作流程**：
```
甲方(CPU) → 找工头老王(APB2) → 填工单(寄存器) → 工人(GPIO)执行
```

## 三、GPIO的8种工作模式

### 输出模式（4种）

| 模式 | 比喻 | 特点 |
|------|------|------|
| **推挽输出** | 甲方说了算 | CPU对引脚有绝对控制权，高低电平都可控制 |
| **开漏输出** | 只能指挥休息 | 只能控制低电平，高电平为高阻态（由外部决定） |
| **复用推挽输出** | 请外部监工 | 由片上外设（如传感器）控制高低电平 |
| **复用开漏输出** | 外部监工+只能休息 | 片上外设控制低电平，高电平高阻态 |

### 输入模式（4种）

| 模式 | 比喻 | 特点 |
|------|------|------|
| **模拟输入** | 专人清点入库 | ADC专用，读取具体电压值（如2.1V、2.2V） |
| **上拉输入** | 仓库默认有货 | 默认高电平，外部输入决定最终状态 |
| **下拉输入** | 仓库默认空 | 默认低电平，外部输入决定最终状态 |
| **浮空输入** | 不清楚库存 | 无默认状态，易受干扰，尽量少用 |

## 四、程序实战：LED闪烁

### 4.1 第一步：找工头老王（开启时钟）

```c
// 开启APB2时钟，使能GPIOA
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);
```

**理解**：先找到老王，让他准备班组A的人。

### 4.2 第二步：填工单（初始化GPIO）

```c
GPIO_InitTypeDef AA;  // 定义工单变量

AA.GPIO_Pin = GPIO_Pin_0;           // 选工人：0号引脚（张三）
AA.GPIO_Mode = GPIO_Mode_Out_PP;    // 工作模式：推挽输出
AA.GPIO_Speed = GPIO_Speed_50MHz;   // 干活速度：50MHz（最快）

GPIO_Init(GPIOA, &AA);  // 把工单交给老王
```

**工单三要素**：
1. **GPIO_Pin**：选哪个工人（Pin_0 ~ Pin_15）
2. **GPIO_Mode**：工作模式（8种之一）
3. **GPIO_Speed**：干活速度（2MHz/10MHz/50MHz）

### 4.3 第三步：下指令（控制输出）

#### 方式一：Set/Reset（吹号/敲锣）

```c
GPIO_ResetBits(GPIOA, GPIO_Pin_0);   // 输出低电平（点亮LED）
Delay_ms(500);                        // 延时500ms
GPIO_SetBits(GPIOA, GPIO_Pin_0);     // 输出高电平（熄灭LED）
Delay_ms(500);                        // 延时500ms
```

**比喻**：吹号=干活（高电平），敲锣=休息（低电平）。

#### 方式二：WriteBit（举旗子）

```c
GPIO_WriteBit(GPIOA, GPIO_Pin_0, Bit_RESET);  // 举旗=干活
Delay_ms(500);
GPIO_WriteBit(GPIOA, GPIO_Pin_0, Bit_SET);    // 放下=休息
Delay_ms(500);
```

**比喻**：举旗=1（高电平），放下=0（低电平）。

#### 方式三：Write（对整个班组操作）

```c
GPIO_Write(GPIOA, 0x0001);  // 对GPIOA整个端口写值
```

**注意**：LED是低电平点亮，需要取反操作。

## 五、硬件连接

```
VCC(3.3V) → 电阻(120Ω) → LED正极(长脚)
                              ↓
                         LED负极(短脚) → GPIOA_Pin_0
```

- **LED点亮条件**：GPIO输出低电平（0V），形成回路
- **加电阻原因**：防止LED太亮烧坏（可选）

## 六、模块化编程技巧

### 使用 #define 重定义

```c
#define LED_PIN    GPIO_Pin_0
#define LED_MODE   GPIO_Mode_Out_PP

// 后续修改只需改这里
AA.GPIO_Pin = LED_PIN;
AA.GPIO_Mode = LED_MODE;
```

**好处**：移植方便，改引脚或模式时不用改一堆代码。

### 创建个人模板

1. 写好基础初始化代码
2. 保存为 `.h` 文件
3. 新项目直接 `#include` 使用

## 七、速查卡片

```
【GPIO操作三步走】
1. 开时钟：RCC_APB2PeriphClockCmd(端口, ENABLE)
2. 填工单：GPIO_InitTypeDef 变量 → 配置Pin/Mode/Speed → GPIO_Init()
3. 下指令：SetBits/ResetBits/WriteBit 控制高低电平

【8种模式速记】
输出：推挽(全控)、开漏(只能低)、复用推挽、复用开漏
输入：模拟(ADC)、上拉(默认高)、下拉(默认低)、浮空(易干扰)

【工地比喻】
CPU=甲方、APB2=工头老王、寄存器=工单、GPIO=工人
```
