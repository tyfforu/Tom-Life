# Day 4 加餐 | 证明型文案专项练习（图文/口播通用）

> [!info] 课程信息
> **第三期 | 自媒体0-1陪跑**

---

## 核心认知

> [!warning] 结构不是听懂就会的，它必须写出来、改出来、练出来

证明型内容本来就是熟练度模型：
- 写1条是偶然
- 写6条才是能力
- 写到10条才会顺手
- 写到20条你才会有"成交感"

---

## 今日任务

> [!success] 交付成果
> - ✅ 1篇完整图文（结构必须完整）
> - ✅ 1条完整口播稿（更口语、能直接念）

**3个硬性标准：**
1. 必须出现结果变化（前后对比/可感知变化）
2. 必须写清过程动作（至少2-3个步骤）
3. 必须有明确承接动作（评论/私信/去主页三选一）

---

## 模板A | 改前改后对比（最强证明结构）

> 成交力最强，直接把"变化"摆在读者面前

### 图文结构

| 模块 | 内容 |
|------|------|
| 场景一句 | 你是不是也在__？（具体处境） |
| 根源一句 | 你卡住不是因为__，而是因为__ |
| 过程三步 | 我做了①—②—③— |
| 对比结果 | 改完后从__变成__ |
| 行动指引 | 你先自查这1点：__ |
| 承接一句 | 需要我按你情况改一版，私信我【__】 |

### 口播结构（60秒）

1. 开头3秒抛场景
2. 点破根源
3. 快速讲三步过程
4. 一句结果对比
5. 明确动作

> [!tip] 练习重点
> 一定要有"前后对比"：数据、时间、反馈、清晰度、效率变化都算。

---

## 模板B | 诊断拆解（让人觉得"被看见"）

> 核心不是"我来教"，而是让对方产生：你真的懂我

### 图文结构

| 模块 | 内容 |
|------|------|
| 场景 | 你现在可能处在__（具体阶段/时刻） |
| 症状 | 你会出现__（具体行为） |
| 诊断 | 问题根源在__ |
| 处方 | 按顺序做①__②__③__ |
| 提醒 | 别做__（一个典型误区） |
| 承接 | 需要我帮你诊断，私信【诊断】 |

### 口播结构（60秒）

场景→症状→原因→动作→提醒→承接

> [!tip] 练习重点
> "症状"一定要具体。不是"你很焦虑"，而是"你每天改3次标题却不敢发"。

---

## 模板C | 清单避坑（高收藏+高转化）

> "避坑"天然具备吸引力

### 图文结构

| 模块 | 内容 |
|------|------|
| 开头 | 新手90%会踩这3个坑：①__②__③__ |
| 解释 | 每个坑解释1句"为什么会错" |
| 做法 | 对应给3个"正确做法" |
| 互动 | 你现在最容易踩的是哪一个？评论告诉我 |
| 承接 | 要我给你一份自查清单，私信【清单】 |

### 口播结构（60秒）

问题→原因→做法→互动→承接

> [!tip] 练习重点
> "坑必须真实"：不是"不要焦虑"，而是"发内容只讲感受，没有具体步骤"。

---

## 模板D | 过程展示（最适合口播）

> 能力感，是靠"过程证明"建立的

### 图文结构

| 模块 | 内容 |
|------|------|
| 开头 | 今天我讲清楚，我怎么把__从__做到__ |
| 第1步 | __（具体动作） |
| 第2步 | __（判断或调整） |
| 第3步 | __（关键升级动作） |
| 结果变化 | 从__变成__ |
| 最小行动 | 如果你也想复制，先做__ |
| 承接 | 私信【__】 |

### 口播结构（60秒）

1. 开头3秒：讲清楚"起点→终点"
2. 正播：用短句讲3个动作
3. 结果强化：再重复一次变化
4. 最小行动
5. 承接动作

> [!tip] 练习重点
> 过程必须具体：不是"我优化了策略"，而是"我把开头从观点改成具体场景"。

---

## 模板E | 案例复盘（信任沉淀型）

> 通过故事展示你的判断能力

### 图文结构

| 模块 | 内容 |
|------|------|
| 故事 | 前几天有个宝子来找我，说她__ |
| 我 | 我一看她正在__ |
| 改 | 我让她先改2个点：①—②— |
| 果 | 改完之后她__（结果变化） |
| 点破 | 一句话点破根源：__ |
| 给 | 你也照做①__，先别着急__ |
| 承接 | 你也卡在这一步的话，私信我【复盘】 |

### 口播结构（60秒）

故事→点破→2个调整→结果→原则→承接

> [!tip] 练习重点
> 故事要真实，变化要清晰：不是"她进步了很多"，而是"从没人问到开始有3个精准咨询"。

---

## 模板F | "你以为 vs 真相"（强冲突开头）

> 开头就制造认知落差

### 图文结构

| 模块 | 内容 |
|------|------|
| 你以为 | __（常见但错误的认知） |
| 真相 | __（一句话直接纠正） |
| 为什么 | __（1句解释底层逻辑） |
| 怎么做 | ①__②__③__ |
| 结果 | __（写清楚变化） |
| 承接 | 私信【__】 |

### 口播结构（60秒）

1. 开头3秒："你以为__？"
2. 紧接："真相是__。"
3. 1句解释为什么
4. 快速讲2-3个步骤
5. 一句话讲结果变化
6. 承接动作

> [!tip] 练习重点
> 冲突必须真实：不是"你以为努力没用"，而是"你以为多写干货就会成交"。

---

## 核心金句

> "别追求完美，先把一条完整跑出来。"

> "一条可能只是尝试，两条开始有感觉，三条之后你就会发现：原来不是不会写，只是之前没有把结构跑熟。"
