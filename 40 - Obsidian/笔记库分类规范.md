## 二、分类逻辑

### 2.1 一级分类（数字前缀）

| 前缀 | 类别 | 说明 | 示例 |
|------|------|------|------|
| **00** | 收集箱 | 新笔记入口，待分类 | 小红书收藏、网页剪藏 |
| **10** | 记录 | 时间维度的个人记录 | 日记、打卡、周报 |
| **20** | 工作与学习 | 输出导向，有目标的学习 | 课程笔记、项目资料、技能学习 |
| **30** | 笔记和灵感 | 输入导向，知识收集 | 读书笔记、健康知识、生活技巧 |
| **40** | Obsidian | 笔记系统本身的配置 | 模板、插件配置 |

### 2.2 分类决策树

```
新笔记
  │
  ├─ 是时间记录（日记/打卡）？ ──→ 10 - 记录/
  │
  ├─ 是学习/工作相关，有输出目标？ ──→ 20 - 工作与学习/
  │   ├─ 嵌入式/硬件相关 ──→ 51单片机/ 或 嵌入式IDE/
  │   ├─ AI 相关 ──→ AI工具/
  │   ├─ 自媒体内容 ──→ 自媒体/
  │   └─ 学习方法/英语/比赛 ──→ 对应子目录
  │
  ├─ 是知识收集/灵感记录？ ──→ 30 - 笔记和灵感/
  │   ├─ AI 知识 ──→ AI相关/
  │   ├─ 书籍笔记 ──→ 读书学习/
  │   ├─ 健康/健身 ──→ 健康养生/ 或 健身增肌/
  │   ├─ 生活技巧 ──→ 实用技巧/
  │   ├─ 个人思考 ──→ 思考感悟/
  │   └─ 待办想法 ──→ wanna do/
  │
  ├─ 是碎片想法？ ──→ 卡片知识/收集箱/
  │
  └─ 是 Obsidian 配置？ ──→ 40 - Obsidian/
```

## 三、工作流程

### 3.1 新笔记入库

```
用户发送内容 → 存入 00 - 收集箱/ → 等待整理指令
```

**存放规则**：
- 小红书内容 → `00 - 收集箱/小红书/`
- 网页剪藏 → `00 - 收集箱/网页剪藏/`
- 其他内容 → `00 - 收集箱/待整理/`

### 3.2 整理指令

用户发出"整理这个笔记"或类似指令后：

1. **读取笔记内容**
2. **判断分类**（使用分类决策树）
3. **清理格式**（统一 frontmatter、标题、结构）
4. **移动到目标目录**
5. **更新 Git**

### 3.3 整理标准

| 维度 | 处理方式 |
|------|----------|
| **Frontmatter** | 统一添加 title、date、tags、source |
| **标题** | 简洁明确，去掉网站后缀 |
| **结构** | 添加目录、摘要表格、callout |
| **链接** | 保留来源链接，添加 `[[wikilink]]` 关联 |
| **标签** | 添加内容标签 + 分类标签 |

## 六、维护规范

### 6.1 定期清理

- **收集箱**：每周清理一次
- **Clippings**：逐步迁移到新分类后删除

### 6.2 重命名规范

- 文件名使用中文，简洁明确
- 去掉网站后缀（如 `-MedSci.cn`）
- 日期格式：`YYYY-MM-DD`

### 6.3 Git 提交规范

```
add: 新增笔记标题
refactor: 笔记标题整理为结构化格式
move: 笔记标题从 A 移动到 B
delete: 删除重复/过期笔记
```
