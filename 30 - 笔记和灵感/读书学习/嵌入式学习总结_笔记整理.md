## 第一块板：精英开发板（F103ZET6，正点原子）

入门级STM32，学习顺序：

### 第一步：裸机开发

- **工具链**：Keil + ST的HAL库（配合CubeMX）
- **核心三件套**：
  - `GPIO` — 最基础的输入输出配置
  - `定时器` — 配置中断，单片机计算的灵魂
  - `串口` — 单片机和PC通讯，最常用的外设
- **常用外设**：OLED/LCD屏幕、ADC（采集电压，电机控制必备）、DAC、DMA（配合ADC绕过CPU传数据）、IIC和SPI（连接各种外设）、485/CAN、红外遥控
- **可选**：TF/SD卡、DSP/FFT

### 第二步：RTOS（FreeRTOS）

- 本质：给单片机加一个实时操作系统
- 学习路径：看手册 → 裸机工程移植FreeRTOS → 学API使用
- 面试可能问知识点，但实际会用+有项目支撑就够了

### 第三步：找项目应用

- 作者因**电机控制项目**自然驱动了深入学习
- 在F1上跑步进电机，发现FLASH不够 → 促成了买第二块板

## 第二块板：电机开发板（F407）

### 电机类型

直流有刷、舵机、无刷直流电机（BLDC）、步进电机

### 核心算法

- **PID** — 最最最常用
- 步进电机加减速算法（梯形/S型）
- 无刷直流电机的**六步换相**和**FOC算法**

### 学术 vs 工程建议

| 方向 | 推荐 |
|------|------|
| 学术研究 | **PMSM的FOC**（有感/无感FOC、DTC、速度双闭环/位置三闭环），文献和项目都很多，值得深学 |
| 工程底层控制 | **直流有刷 + PID**，最常见 |

### 网络通讯（lwIP / TCP-IP）

- 学习路线：lwIP基本概念 → 裸机移植lwIP → Socket API → 服务器通信
- 正点原子视频讲解比较乱，不太推荐看课
- **实际项目更推荐**：STM32+ESP8266 或直接 ESP32（有WiFi，对硬件要求低）

### 单片机阶段小总结

- 学到Linux嵌入式之前，单片机内容基本到此为止
- 进阶方向：LCD图形界面UI（嵌入式也有开源协议）

## 第三块板：STM32MP157

> ⚠️ **建议入门选 Imx6ull**（用的人多、资源多），MP157用的人少、资料难找

### Linux驱动

学习路线：

1. **移植Linux系统**：TF-A安全启动 → Uboot → Linux内核 → 根文件系统
2. **三类驱动**：
   - **字符设备**（最常用）：GPIO、PWM、IIC/SPI、音频、USB、CAN
   - **块设备**：RAM驱动
   - **网络设备**：WiFi/4G外设驱动
   - **IIO驱动**：ADC/DAC

本质上就是比单片机多了"怎么移植操作系统"这一步，其他逻辑一样。

## 几个关键判断

- **F1够用吗？**：学习够用，做项目推荐F4（更大Flash、更强算力）
- **正点原子 vs 野火？**：资料质量差不多，看实验室师兄用什么就跟着用
- **视频 vs 手册？**：裸机看视频；RTOS和lwIP可以直接看手册+源码
- **买不买实际硬件？**：必要。电机控制在脑子里学不动，必须上手跑
- **学完单片机学什么？**：直接上Linux嵌入式，路线和天花板都更高
