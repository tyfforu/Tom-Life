# 嵌入式Linux开发板选择与方向建议

> 来源：知乎问答《嵌入式Linux应该买谁的开发板？》- DeepSeed 回答（281赞）
> 背景：已有51/STM32基础，正在学操作系统，准备入门Linux开发板

---

## 三条推荐路线

### 路线一：STM32F4 + RTOS（推荐有MCU基础的人）

**核心逻辑**：先学RTOS，再上Linux，理解更深。

- 买 STM32F4 系列板子，学习实时操作系统（RTOS）
- 通过配置定时器、PLL、NAND等硬件，对线程/内存/进程通讯/调度有**底层理解**
- 之后再学Linux，学习成本几乎为零，理解比直接从Linux入门的人更强

**就业前景**：
- MCU+RTOS 岗位数量远多于SoC Linux，不是一个数量级
- 应用场景：电机、家电、工控、汽车ECU、手机内置MCU、相机产品等
- 薪资：已普遍 20k~60k+

---

### 路线二：RK开发板 → Linux应用 → Android Framework

**核心逻辑**：用稳定的RK板子专注练Linux应用，为转Android Framework铺路。

- 推荐：韦东山、正点原子、野火的RK板（方案成熟、稳定、便宜）
- 驱动学到**会看厂家代码和设备树**即可，重心放在 **Linux应用开发**
- 把 C++ 学好（有C基础上手没难度），可0成本转 Android Framework

**为什么 Android Framework？**
- 懂的人不多 → 吃香，工资可观
- Linux应用 → Android Framework 几乎零学习成本

**另一出路**：政警军系统（银河麒麟等国产Linux），基本不用担心失业。

> ⚠️ Linux驱动已不值钱：框架固定、厂家技术支持完善、AI辅助开发，薪资没涨甚至跑不赢通胀。

---

### 路线三：买个"问题多多"的板子 → BSP方向

**BSP = 板级支持包**，核心工作：让最小系统/全功能系统在板子上正常跑起来。

**为什么要选问题多的板子？**
- 逼你解决各种 bring up 问题，这才是BSP的核心能力
- 学会看原理图、自己换料、百度/查文档解决问题

**BSP工作阶段**：
1. EVT：基于新硬件移植最小系统，解决所有跑不起来的问题
2. DVT：稳定性测试、高低温老化、性能测试、各类认证

**BSP的坑**：问题千奇百怪，代码闭源，需要改别人写的代码，十分掉头发。

---

## 大方向建议（非嵌入式从业者必看）

> 如果还没入行，建议直接去搞 AI / 大模型 / 机器人算法。

**嵌入式赛道内**的新方向：
- MCU → **RTOS + ROS**
- SoC → **RISC-V（SiFive5）/ NPU驱动**

**未来会被AI替代的岗位**：程序员（普通）、翻译、会计、厨师、快递、外卖、安保、家政、生产加工、驾驶

**不会被替代的岗位**：AI开发、销售采购、金融、医生、律师

---
